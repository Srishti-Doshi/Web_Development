CASE STUDY
State Management in React using Context Providers & Zustand (TypeScript)

--------------------------------------------------
1. Problem Statement
TaskFlow Project Management System

TaskFlow is a project management application where users can:
- Add, edit, and complete tasks from multiple screens
- Switch between light and dark themes
- Access user profile information globally

As the application grows, passing data through props becomes complex
(prop drilling), and frequent state updates cause unnecessary re-renders.

The development team needs a state management solution that is:
- Type-safe
- Scalable
- Easy to test
- Optimized for performance

--------------------------------------------------
2. Objectives

The objectives of this case study are:
- To manage global application state efficiently
- To implement type-safe Context Providers
- To use Zustand for scalable business state
- To avoid unnecessary re-renders and boilerplate
- To maintain clean and maintainable architecture

--------------------------------------------------
3. State Management Strategy

Two approaches are used:

1. React Context Provider
   - Used for global, rarely-changing state
   - Example: Theme (light/dark)

2. Zustand
   - Used for frequently-changing business state
   - Example: Tasks and notifications

This hybrid approach ensures both simplicity and performance.

--------------------------------------------------
4. Theme Management using Context Provider

Theme Context Type Definition

interface ThemeContextType {
  theme: 'light' | 'dark';
  toggleTheme: () => void;
}

Context Creation

const ThemeContext = React.createContext<ThemeContextType | undefined>(undefined);

Provider Implementation

const ThemeProvider = ({ children }: { children: React.ReactNode }) => {
  const [theme, setTheme] = React.useState<'light' | 'dark'>('light');

  const toggleTheme = () =>
    setTheme(prev => (prev === 'light' ? 'dark' : 'light'));

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

Custom Hook for Safe Access

const useTheme = () => {
  const context = React.useContext(ThemeContext);
  if (!context) {
    throw new Error("useTheme must be used within ThemeProvider");
  }
  return context;
};

--------------------------------------------------
5. Using Theme Context in a Component

const ThemeSwitcher = () => {
  const { theme, toggleTheme } = useTheme();

  return (
    <button onClick={toggleTheme}>
      Switch to {theme === 'light' ? 'dark' : 'light'} mode
    </button>
  );
};

Benefits:
- No prop drilling
- Strong type safety
- Easy to use across the app

6. Task Management using Zustand

Task Data Model

interface Task {
  id: string;
  title: string;
  completed: boolean;
}

Store Interface

interface TaskStore {
  tasks: Task[];
  addTask: (title: string) => void;
  toggleTask: (id: string) => void;
}

Zustand Store Definition

import { create } from 'zustand';

const useTaskStore = create<TaskStore>((set) => ({
  tasks: [],
  addTask: (title) =>
    set((state) => ({
      tasks: [
        ...state.tasks,
        { id: Date.now().toString(), title, completed: false }
      ],
    })),
  toggleTask: (id) =>
    set((state) => ({
      tasks: state.tasks.map(task =>
        task.id === id
          ? { ...task, completed: !task.completed }
          : task
      ),
    })),
}));

--------------------------------------------------
7. Using Zustand Store in a Component

const TaskList = () => {
  const tasks = useTaskStore(state => state.tasks);
  const toggleTask = useTaskStore(state => state.toggleTask);

  return (
    <ul>
      {tasks.map(task => (
        <li key={task.id}>
          <input
            type="checkbox"
            checked={task.completed}
            onChange={() => toggleTask(task.id)}
          />
          {task.title}
        </li>
      ))}
    </ul>
  );
};

Benefits:
- No provider required
- Fine-grained re-render control
- Minimal boilerplate
- Excellent TypeScript support

--------------------------------------------------
8. Combining Context and Zustand

- Context is used for application-level settings (theme)
- Zustand is used for business logic (tasks)
- Both systems work independently and cleanly
- Improves scalability and maintainability

--------------------------------------------------
9. Best Practices Followed

- Strong typing for all state and actions
- Immutable state updates
- Custom hooks for context usage
- Selector pattern in Zustand to reduce re-renders
- Separation of concerns (UI vs state)

--------------------------------------------------
10. Conclusion

This case study demonstrates effective state management in React using
Context Providers and Zustand with TypeScript.

By combining both approaches:
- Global settings remain simple
- Business state remains performant
- Type safety is enforced throughout the application
- The codebase stays scalable and maintainable

This architecture is suitable for both small and large React applications.
