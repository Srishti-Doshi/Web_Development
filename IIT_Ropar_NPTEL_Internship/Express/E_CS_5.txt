Title: Routing Controllers, Middleware, and Request Validation in Express
Case Study: Crumb & Craft Bakery


------------------------------------------------------------
1. Problem Statement: The Overwhelmed Bakery

Crumb & Craft Bakery started as a small neighborhood shop. Orders were simple and handwritten. Everyone knew who handled what, and mistakes were rare.

However, after gaining popularity across the city, things became chaotic:

• Orders were written on sticky notes and sometimes got lost.  
• Bread orders accidentally went to the cake team.  
• Allergy information was ignored during rush hours.  
• Inventory was not checked before confirming orders.  

The worst incident happened during a holiday rush when 500 online orders were placed within one hour.

• A child with a nut allergy nearly received a peanut butter cake.  
• Customers were allowed to order more items than were in stock.  
• Some customers received confirmation emails, others did not.  

The bakery realized that the problem was not lack of effort — it was lack of structure.

The challenge was to redesign the system so that:

• Every request goes to the correct team.  
• Errors are caught before processing begins.  
• Customers always receive clear and consistent responses.

------------------------------------------------------------
4. Understanding Controllers

A controller is simply a module that groups related routes together.

Without Controllers (Flat Structure):

app.post("/orders", ...);
app.get("/orders/:id", ...);
app.post("/inventory", ...);
app.get("/inventory/:item", ...);

All logic is mixed together in one place.

With Controllers (Modular Structure):

OrderController handles all /orders routes.  
InventoryController handles all /inventory routes.  

This makes the system organized and easier to maintain.

5. Step-by-Step Implementation

A. Define the Order Data Structure

Instead of handwritten notes, we define a clear structure.

interface Order {
  id: string;
  customerName: string;
  flavor: string;
  quantity: number;
  pickupDate: string;
}

This ensures:

• Every order has an ID  
• Flavor is always specified  
• Quantity is defined  
• Pickup date is never missing  


------------------------------------------------------------
B. Store Orders Safely

const orders: Order[] = [
  {
    id: "1",
    customerName: "Maria",
    flavor: "vanilla",
    quantity: 2,
    pickupDate: "2024-07-10"
  }
];

Now all orders are digitally stored instead of written on paper.


------------------------------------------------------------
C. Creating the Order Controller

import { JsonController, Get, Post, Param, Body } from "routing-controllers";

@JsonController("/orders")
export class OrderController {

  @Get("/")
  getAll() {
    return orders;
  }

  @Get("/:id")
  getOne(@Param("id") id: string) {
    const order = orders.find(o => o.id === id);

    if (!order) {
      return { status: "error", error: "Order not found" };
    }

    return { status: "success", data: order };
  }

  @Post("/")
  create(@Body() order: Omit<Order, "id">) {
    const newOrder: Order = {
      ...order,
      id: (orders.length + 1).toString()
    };

    orders.push(newOrder);

    return { status: "success", data: newOrder };
  }
}

Explanation:

• @JsonController("/orders") groups all order routes.  
• @Get("/") returns all orders.  
• @Get("/:id") returns a specific order.  
• @Post("/") creates a new order and assigns a unique ID.  

Now all order-related logic is handled in one place.

6. Middleware – Adding Checkpoints

Middleware works like quality checks before baking starts.

Example: Allergy Check Middleware

import { ExpressMiddlewareInterface, Middleware } from "routing-controllers";

@Middleware({ type: "before" })
export class AllergyMiddleware implements ExpressMiddlewareInterface {

  use(req: any, res: any, next: any) {
    const { flavor } = req.body;

    if (flavor && flavor.toLowerCase().includes("peanut")) {
      throw new Error("Peanut allergy alert");
    }

    next();
  }
}

This prevents dangerous orders from being processed.


------------------------------------------------------------
7. Request Validation

Validation ensures that orders are complete and correct before being accepted.

Using class-validator:

import { IsString, IsInt, Min } from "class-validator";

class CreateOrderDTO {

  @IsString()
  customerName: string;

  @IsString()
  flavor: string;

  @IsInt()
  @Min(1)
  quantity: number;

  @IsString()
  pickupDate: string;
}

@Post("/")
create(@Body({ validate: true }) order: CreateOrderDTO) {
  // This runs only if validation passes
}

If validation fails, the system automatically returns an error.

This prevents:

• Missing fields  
• Negative quantities  
• Invalid formats  


------------------------------------------------------------
8. Data Flow (How Everything Works Together)

1. Request comes in  
2. Middleware runs (allergy, inventory checks)  
3. Validation runs  
4. Controller handles business logic  
5. Response is sent back  

This layered approach prevents chaos.

10. Common Pitfalls & Best Practices

Pitfall:
Putting all routes in one file.

Best Practice:
Use controllers to group related routes.

Pitfall:
Skipping validation.

Best Practice:
Always validate user input before processing.

Pitfall:
Inconsistent response formats.

Best Practice:
Return consistent JSON structure.
